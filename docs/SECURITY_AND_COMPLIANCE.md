# 安全与合规说明

本文不是法律意见，只记录 UseChat 的产品和工程边界。

## 定位

UseChat 是本地 Computer Use 工具。它在用户显式授权后，操作用户自己电脑上可见的消息软件。

可以这样说：

- 本地优先；
- 可见 Computer Use；
- 用户授权；
- 个人助理；
- Agent 使用你自己的桌面软件。

不要这样说：

- 防封；
- 绕过检测；
- 群发；
- 群控；
- 营销自动化；
- 无限账号；
- 协议破解。

## 技术红线

- 不逆向协议。
- 不注入进程。
- 不 patch 客户端。
- 不读取本地数据库。
- 不扫描缓存来推断消息或附件。
- 没有用户策略时不隐藏后台发送。
- 不把 API key 写入日志。
- CLI 输出不打印原始密钥。

## 隐私默认值

- 截图只用于当次模型调用，默认不保存。
- 剪贴板内容不写日志。
- OCR 全文默认不写日志。
- 附件留在本机，除非用户或嵌入平台显式传输。
- 诊断导出必须显式触发，并支持脱敏。

## 发送策略

CLI 默认应在发送前确认目标对话和内容。非交互使用必须通过 `--yes` 或显式策略选择。

如果窗口、聊天标题、输入框、剪贴板或用户活动状态不确定，UseChat 必须 fail closed。
